Documentation

The web application will provide users with access to informnation about different movies, directors and genres. Users will be able to sign up, update their personal info and create a list of their favorite movies.


Entities Table

ENTITY ATTRIBUTES
Movies ID, Title, Description, Genre, Director, URL to image, Featured (Y/N)
Director ID, Name, Bio, Birth year, Death year
Genre ID, Name, Description
Users ID, Username, Password, Email, Birthday, Favorite movies



Request URL HTTP Method Request body data format Respond body data format
Return a list of ALL movies to the user /movies GET none A JSON object holding data about all the movies
Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user /movies/[Name] GET none A JSON object holding data about a single movie, containing description, genre, director, image URL,
{
    id: "title",
    genre: {
            Thriller,
            Comedy,
            Action,
            Drama,
    }
    director: "name",
    image URL: "link to image URL"
}
Return data about a genre (description) by name/title (e.g., “Thriller”) /movies/[GENRE] GET none A JSON object holding data about a movie genre
{
    name: "thriller",
    description: "link top text"
}
Return data about a director (bio, birth year, death year) by name /movies/[DIRECTOR] GET none A JSON object holding data about a movie director
{
    name: "Matthew Vaughn",
    bio: "link to text",
    birthyear: 1971,
    death year: "still alive"
}
Allow new users to register /users POST A JSON object holding data about a new user to add, structured like
{
    name: “Paula Gomez”,
    username: “paula”,
    email: “pauline@yahoo.com”
} 		A JSON object holding data about a new user
{
    id: 23,
    name: “Paula Gomez”,
    username: “paula”,
    email: “pauline@yahoo.com”
}
Allow users to update their user info (username) /users PUT NONE A text message indicating that the user has their user info.
Allow users to add a movie to their list of favorites (showing only a text that a movie has been added—more on this later) /users/[FAVS] POST NONE A text message indicating that the user has added a new movie to the favorite list
Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed—more on this later) /users/[FAVS] DELETE NONE A text message indicating that the user has removed a movie from the favorites list
Allow existing users to deregister (showing only a text that a user email has been removed—more on this later) /users DELETE NONE A text message indicating that the user has deregistered from myFlix
